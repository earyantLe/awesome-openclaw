# Resource Submission Guide

Thank you for wanting to contribute to Awesome OpenClaw! This guide will help you submit resources effectively.

## What Can You Submit?

### ✅ Accepted Resource Types

- **Skills** - Custom OpenClaw skills you've built
- **Extensions** - Plugins or integrations
- **Projects** - Applications built with OpenClaw
- **Tutorials** - Blog posts, videos, or guides
- **Tools** - Utilities that complement OpenClaw
- **Documentation** - Helpful docs or translations
- **Blogs/Newsletters** - OpenClaw-related content

### ❌ Not Accepted

- Self-promotion without genuine value
- Broken or dead links
- Duplicate entries
- Unrelated resources
- Spam or low-quality content
- Discriminatory or harmful content

## How to Submit

### Option 1: Pull Request (Recommended)

1. **Fork** the repository
2. **Create** a branch: `git checkout -b feature/add-my-resource`
3. **Edit** `README.md` and add your resource to the appropriate section
4. **Follow** the existing format (see examples below)
5. **Commit**: `git commit -m "Add: [Your Resource Name]"`
6. **Push**: `git push origin feature/add-my-resource`
7. **Open** a Pull Request

### Option 2: GitHub Issue

If you're not comfortable with Git:

1. Go to the [Issues](https://github.com/earyantLe/awesome-openclaw/issues) page
2. Click "New Issue"
3. Use the "Suggestion" template
4. Provide all required details

## Format Examples

### Skills

```markdown
| `skill-name` | Author | Description |
|--------------|--------|-------------|
| `my-skill` | Your Name | Brief description (1-2 sentences) |
```

### Extensions

```markdown
| [Extension Name](https://url) | Description | Author |
|-------------------------------|-------------|--------|
| [My Extension](https://github.com/user/repo) | What it does | @username |
```

### Projects

```markdown
| [Project Name](https://url) | Author | Description |
|-----------------------------|--------|-------------|
| [My Project](https://github.com/user/repo) | Your Name | What it does |
```

### Tutorials

```markdown
| Title | Author | Description |
|-------|--------|-------------|
| [My Tutorial](https://url) | Your Name | What readers will learn |
```

## Quality Standards

### Required

- ✅ Resource must be related to OpenClaw ecosystem
- ✅ Working URL (if applicable)
- ✅ Clear, concise description (1-2 sentences)
- ✅ Proper markdown formatting
- ✅ Placed in correct section

### Recommended

- ⭐ Unique value (not already covered)
- ⭐ Well-maintained (regular updates)
- ⭐ Good documentation
- ⭐ Active community/support
- ⭐ Tested and working

## Review Process

1. **Submission**: You submit a PR or issue
2. **Review**: Maintainers review within 7 days
3. **Feedback**: We may request changes or clarifications
4. **Merge**: Once approved, your resource is added
5. **Announcement**: Featured in next community update

### Review Timeline

- **PR Review**: Within 7 days
- **Merge**: Within 14 days (if no issues)
- **Rejection**: Immediate with feedback

## Tips for Acceptance

1. **Search First**: Check if the resource already exists
2. **Test Links**: Make sure all URLs work
3. **Be Specific**: Clear descriptions help users
4. **Follow Format**: Match existing entries
5. **English Descriptions**: Or provide English translation

## Common Rejection Reasons

- ❌ Duplicate of existing resource
- ❌ Broken or dead link
- ❌ Vague or unclear description
- ❌ Not related to OpenClaw
- ❌ Self-promotion without value
- ❌ Poor formatting
